Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attribute defines additional information about an element and is always defined in the beginning of the element.
  • Metadata is data about a webpage that a browser uses to effectively serve the page to users.
  • Inside the < head > element, you can use the metadata tag < title > to define the page. This is required and important for SEO (Search Engine Optimization).
  • Semantic HTML elements describe its contents and the role is plays on a web page. This makes html easier to read and plays an important role in accessibility for a wide range of users. < body > is an example of this.
  • The words “element” and “tag” are commonly used interchangeably in the web development world, but it is important to know the difference. An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.

CSS

  • There are 3 ways to style a webpage using CSS.
    • Inline CSS: these apply to a specific HTML tag, using a style attribute to apply style to a particular page element.
    • Internal style sheet: contains CSS rules for the page in the head section of the HTML file. The rules will only apply to that particluar page.
    • External style sheet: preferred in most cases. Allows devs to keep all CSS rules in a seperate file & makes changes easier. Used by placing a file via link placed in the HTMLs head.
  • Declarations contain two components: the CSS property and the value of that property. Example: color:blue;
  • Margin property acts like a margin in a word doc. It applies to the outside of an element.
  • Padding adds spaces around the content inside an element. When padding is increased, so is the size of the element.
  • * will apply a rule to all elements visible on the page. Using this declares all will have the same style properties.
  • DRY code, or Do Not Repeat Yourself, is a principle that encourages devs to reuse or share pieces of code to reduce the number of lines and complexity in a codebase.

Git

  • Github Jargon
    • Repo or repository is a folder for your project that contains all files and their revision history.
    • Branch is a workspace where you can make changes that don't affect your main branch or live site.
    • Markdown or .md is the way content files are formatted in Github.
    • Commit changes is saving a new version of a file you are working on; adding changes to your branch.
    • Pull request, or PR, is asking for changes on a branch to merge into another branch.
    • Merge is when a PR is approved, changes are "pulled in" or merged into the main branch.
    • Issue is a way to report a new tak or content fixes; track progress.

  • Git commands
    • git checkout -b branchname create a new branch
    • git status check your current branch
    • git add -A stage all files (always stage after making changes!)
    • git commit -m "message" descriptive message to keep track of commits
    • git pull origin main pull source code from github to ensure you have the latest commits and changes
    • git push branchname push your current branch up
